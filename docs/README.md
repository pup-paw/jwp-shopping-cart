## 목표

### 공통

- 프론트와 협업을 하며 API 를 작성할때 어떤 부분을 중심적으로 논의해야 할지 고민하고, 기록해두기
- 항상 의식적으로 근거를 가지고 코드 짜기

### 매트

- 현재 알고 있는 Spring MVC에 대한 구조를 다시 한 번 인지하며 해당 기술을 적절히 활용하기
- 처음부터 너무 과하게 설계하지 않기
- MVC 를 다시 한번 공부하며 페어의 이해를 돕기 위해 노력하기

### 야호

- ATDD 를 중심으로 개발하기
- auth 에 대한 대략적인 흐름을 파악해보기
